# File Catalog

A reference for what each file in the repository is responsible for. Grouped by directory; one short paragraph per file. For deeper detail on any subsystem see [architecture.md](./architecture.md), [components.md](./components.md), [ipc-protocol.md](./ipc-protocol.md), and [viz-engine.md](./viz-engine.md).

## Repository root

| File | Purpose |
|---|---|
| `package.json` | Project manifest. Declares dependencies, the `bun run` script aliases (`dev`, `build`, `preview`, `dist`, `lint`, `fmt`, `fmt:check`, `typecheck`), the `productName`/`version` consumed by electron-builder, and the full `build` block (targets, code signing, `vPrefixedTagName`, GitHub publish channel, `artifactName` template). |
| `bun.lock` | Locked dependency graph for reproducible installs. Regenerated by `bun install`. |
| `electron.vite.config.ts` | electron-vite build config for the three processes (main / preload / renderer). Defines the `@/` and `@shared/` path aliases, manual chunk splits (`@soniox/client`, `solid-js`), and Tailwind/Solid plugins. |
| `tsconfig.json` | Root TypeScript project that fans out to the app and node configs. |
| `tsconfig.app.json` | TS config for the renderer (DOM lib, Solid JSX, `@/`/`@shared/` paths). |
| `tsconfig.node.json` | TS config for the main and preload processes (Node libs, no DOM). |
| `lefthook.yml` | Pre-commit hook configuration. Runs `oxlint`, `oxfmt --check`, and `tsc --noEmit` on staged files. |
| `LICENSE` | License terms. |
| `README.md` | High-level project overview, feature list, getting-started, configuration table, project structure, and release flow. |
| `CLAUDE.md` | Local-only AI assistant instructions (gitignored). |

## `vizengine/` (Vizrt scene scripts)

These are reference scripts that run *inside* Viz Engine (not in the Electron app). They live in the repo so the Vizrt-side logic is versioned alongside the controller that drives it.

| File | Purpose |
|---|---|
| `main.viz` | Script attached to the `SCROLL_ROOT` container in the broadcast scene. Drives the per-field scroll/reset behavior the app's controller targets. |
| `pause_play.viz` | Script on the `WAIT_MESSAGE` container handling pause/resume of the scroll animation. |

## `src/main/` (Electron main process)

| File | Purpose |
|---|---|
| `index.ts` | Main-process entry point. Enforces single-instance lock, wires up app lifecycle (`ready`/`window-all-closed`/`activate`), loads config, creates the BrowserWindow, registers IPC handlers, and starts the Viz Engine controller, metrics collector, and updater. |
| `window.ts` | BrowserWindow creation and renderer-crash recovery. Configures `sandbox: false` (required â€” see CLAUDE.md), preload script, and reload-on-crash logic. |
| `config.ts` | Default `AppConfig` and validation. Bridges `store.ts` to the rest of the main process; exposes `getConfig()` / `saveConfig()` with shape validation. |
| `store.ts` | `electron-store` wrapper. Persists `AppConfig` and the Soniox API key (encrypted via `safeStorage` / OS keychain). |
| `ipc.ts` | Central `ipcMain.handle` registration for every channel exposed to the renderer (config, API key, session, mic, perf, viz, updater). The single source of truth for the IPC surface. |
| `session.ts` | Session lifecycle: opens/closes the timestamped session log file, buffers feed-file writes (200 ms flush interval), and writes `feed.txt` atomically via `tmp` + `rename`. |
| `logger.ts` | Tiny structured JSON logger used everywhere in the main process. Emits `{ ts, level, tag, ...data }` lines to stdout/stderr. |
| `metrics.ts` | Periodic perf snapshot collector. Samples per-process CPU/memory via `app.getAppMetrics()`, main-process RSS/heap, and event-loop lag, then pushes `PerfSnapshot` to the renderer. |
| `updater.ts` | electron-updater integration. Schedules background update checks every 4 hours and forwards `update-status` events to the renderer. |
| `viz-engine.ts` | Viz Engine TCP controller. Owns the persistent command socket and the dedicated scroll socket, scene loading, slot writes, scroll/pause animation, idle auto-pause, hard reset, scene-state polling, the `vizReconnect()` manual trigger, and `vizTestConnection()` one-shot probe. |

## `src/renderer/`

### `index.html`

The renderer entry HTML. Loads `theme-init.js` synchronously (to apply the persisted theme before first paint and avoid a flash) and bootstraps `src/index.tsx`.

### `public/theme-init.js`

Pre-paint theme bootstrap. Reads `localStorage.theme` and sets `data-theme` on `<html>` before any CSS is parsed, eliminating the dark/light flash on startup.

### `src/`

| File | Purpose |
|---|---|
| `index.tsx` | Solid renderer entry. Mounts `<App />` into `#root` and registers any global error handlers. |
| `App.tsx` | Root component. Owns top-level state (config, session running, settings modal open, viz status, toasts, perf monitor toggle), wires keyboard shortcuts (Ctrl+Space, Ctrl+Shift+P), composes the three-pane layout (Speech / Translation / Viz) with resize handles, and acts as the conduit between Soniox events and the entry manager. |

### `src/lib/`

| File | Purpose |
|---|---|
| `ipc.ts` | Typed wrapper around `window.electronAPI`. Every method the renderer calls into main goes through here, plus the `on*` helpers that return unsubscribe functions. |
| `types.ts` | Renderer-only types (`TranscriptEntry`, `TranslationEntry`, `EntryStatus`) and re-exports of shared types so renderer code can import everything from `@/lib/types`. |
| `soniox.ts` | Soniox SDK integration. Creates the `SonioxClient`, configures the `MicrophoneSource`, sets `auto_reconnect`/`max_reconnect_attempts`/`reconnect_base_delay_ms`, accumulates final/non-final tokens, dispatches transcript/translation/state callbacks, batches per-line log writes, and surfaces user-facing error messages. |
| `entry-manager.ts` | Reactive store driving the Translation pane. Manages pending/editing/confirmed/sent state, the per-entry review-time countdown (auto-confirm), word/latency/STT counters, and the rolling cap (`MAX_ENTRIES`). |
| `audio-level.ts` | Web Audio analyser shared by all waveform consumers. `startAudioLevel(deviceId)` opens a media stream, `getLevel()` returns a 0â€“1 RMS sample with a silence threshold. |
| `use-auto-scroll.ts` | Solid hook that pins a scroll container to the bottom unless the user scrolls up; respects an optional `paused` accessor. |
| `perf.ts` | Reactive perf monitor used by `PerfOverlay`. Subscribes to main-process `PerfSnapshot` pushes, measures FPS via `requestAnimationFrame`, and pings IPC RTT. |
| `errors.ts` | App-wide error reporter. `reportError(category, message, original)` routes to toast UI and console; `onAppError` lets components subscribe. Defines the `ErrorCategory` union (`mic`/`api-key`/`session`/`config`/`network`/`viz`/`unknown`). |
